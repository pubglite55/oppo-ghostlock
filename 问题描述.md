# 项目问题梳理

## 项目起源

### 项目背景
GhostLock (CVE-2026-43499) 是一个影响 Linux 2.6.39 至 7.1-rc1 的内核栈 UAF 漏洞，通过 `FUTEX_CMP_REQUEUE_PI` 竞态条件触发。NebuSec/CyberMeowfia 已发布 x86_64 平台的 exploit，但无法直接在 ARM64 设备上运行。

### 最初要解决的核心问题
将 GhostLock exploit 从 x86_64 适配到 OPPO Find N2 (ARM64, kernel 5.10.236)，实现无 root 环境下的内核提权。

## 核心问题清单

### 已解决问题

| # | 问题 | 解决方案 |
|---|------|----------|
| 1 | KernelSnitch mm_struct 泄漏失败 | 7-bug 修复 (IDENTITY range, KSNITCH_COLLISIONS, MM_STRUCT_SZ, hashsize, pile-up, futex_hash, nr_cpu_ids) |
| 2 | slide pselect crash | PR #13 bypass — 绕过 slide，直接计算 kaslr_base |
| 3 | kaslr_base/text_addr 架构性错误 | 分离 kaslr_base (text base) 和 direct_map_base |
| 4 | NDK 编译路径问题 | 使用 NDK r29 + android35 API |
| 5 | target.h 偏移错误 | IDA Pro 全量偏移验证 (70+ 偏移) |
| 6 | CVE-2026-23274 适配 | ARM64 ROP gadgets 提取，但触发需要 CAP_NET_RAW |

## 问题解决状态

### 已解决 (6 个)

1. **KernelSnitch mm_struct 泄漏** — 通过 7-bug 修复解决，已验证工作
2. **slide pselect crash** — 通过 PR #13 bypass 解决，直接计算 kaslr_base
3. **kaslr_base 架构性错误** — 修复 text_addr() 计算，使用正确的内核文本基地址
4. **NDK 编译问题** — 使用 NDK r29 + android35 API
5. **target.h 偏移错误** — IDA Pro 全量偏移验证
6. **CVE-2026-23274 适配** — ARM64 ROP gadgets 提取完成，但触发需要 CAP_NET_RAW

### 待解决 (6 个)

1. **内核写原语缺失** — 核心阻塞，需要新的漏洞或攻击路径
2. **pselect fd_set 间隙** — 架构性问题，无法通过参数调优解决
3. **configfs 不支持** — 内核配置问题，无法绕过
4. **设备无 root** — production build 限制
5. **SELinux 阻止** — shell 域策略限制
6. **user namespaces 未启用** — 内核配置限制
